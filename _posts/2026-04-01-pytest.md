# Pytest 测试框架 系统学习大纲

## 一、Pytest 概述
*   **1.1 Pytest 简介**
    *   什么是 Pytest
    *   Pytest 的特点和优势
    *   与其他测试框架对比（unittest, nose）
    <details markdown="1">
    <summary>详情关于pytest和unittest对比</summary>

    | 对比维度 | pytest | unittest |
    | :--- | :--- | :--- |
    | **设计哲学** | 更灵活、简洁，遵循“约定优于配置”原则。 | 基于 Java 的 JUnit 框架，采用经典的 xUnit 风格，结构严谨。 |

    | **编写风格** | 使用普通的 Python 函数和 `assert` 语句，代码更简洁。 | 必须继承 `unittest.TestCase` 类，并使用 `self.assertXxx()` 系列方法。 |
    | **断言方式** | 直接使用 Python 原生的 `assert` 语句，失败时信息更清晰易读。 | 使用 `self.assertXxx()` 方法（如 `assertEqual`, `assertTrue`）。 |
    | **夹具管理** | 通过 `@pytest.fixture` 装饰器提供强大、灵活的夹具系统，支持作用域和自动使用。 | 通过 `setUp`/`tearDown`（方法级）和 `setUpClass`/`tearDownClass`（类级）管理。 |
    | **测试发现** | 自动发现以 `test_` 开头的函数和以 `Test` 开头的类中的 `test_` 方法。 | 自动发现继承 `unittest.TestCase` 的类中的 `test_` 开头的方法。 |
    | **插件生态** | 拥有极其丰富和强大的插件生态系统（如并发执行、HTML报告、覆盖率集成等）。 | 生态相对简单，功能扩展主要依赖原生模块或少量第三方库。 |
    | **执行控制** | 命令行功能强大，支持通过 `-k` 筛选用例、`-m` 标记用例、`--lf` 只运行上次失败的用例等。 | 命令行功能相对基础。 |
    | **报告输出** | 默认输出简洁，失败时提供非常详细的上下文差异分析。 | 输出为标准 xUnit 风格报告。 |
    | **与unittest兼容** | **可以无缝运行** unittest 测试用例，无需修改代码。 | 无法直接运行 pytest 风格的测试函数。 |

    </details>

## 二、测试发现与执行
*   **2.1 测试发现规则**
    *   文件命名规则：test_*.py 或 *_test.py
    *   函数命名规则：test_*
    *   类命名规则：Test*
    *   方法命名规则：test_*

*   **2.2 执行测试的方式**
    *   运行所有测试
    *   运行指定模块
    *   运行指定目录
    *   运行指定测试函数

    ```
    # 运行所有测试
    pytest

    # 运行指定文件
    pytest test_module.py

    # 运行指定测试函数
    pytest test_module.py::test_function

    # 运行指定类
    pytest test_module.py::TestClass

    # 运行指定类中的方法
    pytest test_module.py::TestClass::test_method
    ```

*   **2.3 常用命令行选项**
    *   -v: 详细输出
    *   -q: 简洁输出
    *   -s: 显示 print 输出
    *   -k: 关键字过滤
    *   -m: 标记过滤
    *   -x: 遇到失败立即停止
    *   --maxfail=n: 指定最大失败数
    *   --collect-only: 收集但不执行
    *   --tb=style: 设置回溯显示样式

    ```
    pytest -v -s
    pytest -k "login"
    pytest -m "slow"
    pytest --maxfail=2
    ```

*   **4.2 基本用法**
    *   @pytest.fixture 装饰器
    *   Fixture 参数
    *   使用 Fixture

    ```
    import pytest

    @pytest.fixture
    def sample_data():
        return {"name": "Alice", "age": 30}

    def test_user_data(sample_data):
        assert sample_data["name"] == "Alice"
        assert sample_data["age"] == 30
    ```

*   **4.3 Fixture 作用域**
    *   function: 函数级别
    *   class: 类级别
    *   module: 模块级别
    *   session: 会话级别

    ```
    @pytest.fixture(scope="session")
    def db_connection():
        # 建立数据库连接
        conn = create_connection()
        yield conn
        # 清理
        conn.close()

    @pytest.fixture(scope="module")
    def setup_data():
        return {"key": "value"}
    ```

*   **4.4 Fixture 自动使用**
    *   autouse=True
    *   全局 Fixture

    ```
    @pytest.fixture(autouse=True)
    def setup_each_test():
        print("\n=== 开始测试 ===")
        yield
        print("\n=== 结束测试 ===")
    ```

*   **4.5 Fixture 之间的依赖**
    *   Fixture 调用其他 Fixture
    *   依赖关系

*   **4.6 内置 Fixture**

## 八、Mock 和 Monkeypatch
*   **8.1 什么是 Mock**
    *   模拟对象
    *   测试替身
    *   使用场景

*   **8.2 unittest.mock**
    *   Mock 对象
    *   MagicMock
    *   patch 装饰器

    ```
    from unittest.mock import Mock, patch

    def test_mock():
        mock_obj = Mock()
        mock_obj.method.return_value = 42
        assert mock_obj.method() == 42
        mock_obj.method.assert_called_once()

    @patch("module.function")
    def test_patch(mock_function):
        mock_function.return_value = "mocked"
        result = module.function()
        assert result == "mocked"
    ```

*   **8.3 pytest-mock 插件**
    *   mocker Fixture
    *   简化 Mock

    ```
    def test_with_mocker(mocker):
        mock_func = mocker.patch("module.function")
        mock_func.return_value = "mocked"
        result = module.function()
        assert result == "mocked"
        mock_func.assert_called_once()
    ```

*   **8.4 monkeypatch**
    *   临时修改
    *   环境变量
    *   属性修改

    ```
    import os

    def test_env_var(monkeypatch):
        monkeypatch.setenv("API_KEY", "test_key")
        assert os.getenv("API_KEY") == "test_key"

    def test_attribute(monkeypatch):
        class MyClass:
            value = 1
        
        obj = MyClass()
        monkeypatch.setattr(obj, "value", 2)
        assert obj.value == 2
    ```
